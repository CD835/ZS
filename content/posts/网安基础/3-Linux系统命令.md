**环节一：了解操作系统**

**什么是操作系统？**

操作系统*（缩写：OS）*是一种**手机、电脑**内置的程序，用来协作各种硬件，以与用户进行交互。

**生活中有哪些操作系统？**

电脑常见有： Windows 、开源的 Linux【macOS、kali】、华为鸿蒙系统。

手机常见有：Android（谷歌开发）、iOS（苹果公司开发）。

**Linux的目录结构？**

Linux系统中的目录有以下**特点**：

- **目录结构像一颗倒挂的树**（只有一个盘）

- **/   是所有路径的顶点**，即操作系统的启示路径根目录

- **地址描述稍微特别**，比如： /root/data/a.txt，表示根目录下的root文件夹内有data文件夹，内有a.txt

**二 、命令行操作指令**

在Kali终端中，使用命令行是非常重要的。

**1、系统管理指令**

- **sudo su**   ： 输入密码的时候是看不到的 
  - 用途：用于获取管理员（root）权限，以便执行一些需要更高权限才能完成的系统操作，例如修改系统关键配置文件、安装特定软件等。在很多基于 Linux 的系统中，普通用户权限受限，通过此命令切换到超级用户模式，后续操作将具备管理员的全部权力。不过使用时需谨慎，不当操作可能导致系统故障。

- **apt-get**   ：python里边软件版的管理工具叫 pip
  - 用途：是一款功能强大的软件包管理工具，用于软件包的安装、更新以及日常管理。
  
  - **语法及示例：**
  - **apt-get update**  ：执行该命令可更新本地软件包列表，使其与软件源服务器上的最新信息同步。这是安装新软件包或升级已有软件包前的必要步骤，能确保获取到最新的软件版本信息，如在安装新软件前先运行apt-get update。
      - 如果apt-get update不成功，根据https://note.youdao.com/s/YXEdcwDl教程操作即可
    
  - apt-get install 包名：用于安装指定的软件包，只需将想要安装的软件包名称替换 “包名” 即可，例如apt-get install firefox，就能安装火狐浏览器，或者apt-get install vim，能够安装vim编辑器。
    
  - apt-get upgrade   ：此命令用于升级系统中已安装的软件包，它会依据之前更新的软件包列表，将符合升级条件的软件包升级到最新版本，保持系统软件处于较新状态，提升系统性能与安全性，比如定期运行sudo apt-get upgrade维护系统。
  
- ifconfig ：

  - 用途：用于显示系统所有网络接口的详细信息，包括但不限于网络接口的名称（如 eth0、wlan0 等常见接口名）、IP 地址（IPv4 和 IPv6 地址）、子网掩码、MAC 地址以及网络接口的状态（是否启用、连接情况等）。这对于排查网络连接问题、配置网络参数等网络相关操作至关重要，比如网络连接异常时，输入ifconfig查看网络接口是否获取到正确的 IP 地址。

- ping ：

  - 用途：向指定的IP地址或域名发送网络请求，用于测试网络连接。

    - IP地址：电脑的身份证；

    - 域名：名字

- reboot：
  - 用途：用于立即重启系统，在系统更新软件、修改关键配置后可能需要重启以使更改生效，或者遇到系统死机、性能严重下降等故障情况，通过此命令强行重启系统恢复正常运行。不过重启操作应确保当前系统工作已妥善保存，避免数据丢失。

- shutdown -h now  ：
  - 用途：用于立即关闭系统，通常在不需要系统继续运行，如完成当天工作、进行硬件维护等场景下使用，与reboot类似，执行前需确保数据保存妥当，防止因突然关机造成数据损失。同时，它比普通关机操作更加直接快速，满足紧急关闭系统的需求。

**3、网络扫描与分析指令**

1. nmap：

   - 用途：一款功能强大的网络扫描工具，主要用于发现**网络中**的主机以及主机上开放的服务，助力网络管理员或安全人员了解网络态势。
   - ip是电脑在互联网上的唯一标识。 ip通常是对应一台电脑的。那么在电脑中有许多的程序，为了对这些程序进行更好的数据管理，电脑会给每个程序分配一个端口，这个端口相当于程序的一个编号。
   - 端口 是固定的 还是变化的？
   - 语法及示例：
     - nmap  目标 IP：对目标 IP 地址展开基本的网络扫描操作，能初步探测出目标主机开放的端口，知晓哪些网络服务可能正在对外提供服务，如nmap 192.168.1.100。

- nmap -sS 目标 IP：运用 SYN 扫描方式对目标 IP 地址实施端口扫描。这种扫描方式较为隐蔽，通过发送 SYN 包模拟 TCP 连接建立过程的三步握手，能快速且相对隐蔽地探测端口开放情况，例如nmap -sS 192.168.1.100。

2.**netcat（nc）**：

- 用途：是一款极为多功能的网络工具，在网络连接、数据传输等诸多场景广泛应用，既可以作为服务器端监听端口等待连接，也能作为客户端主动连接目标端口。

- 语法及示例：

  - nc -lp 端口：作为服务器端，在指定端口上进行监听，准备随时接收外来的连接请求，例如nc -lp 8080，此时该主机的 8080 端口处于监听状态，等待其他主机连接。

**4、目录管理指令**

**pwd：**

- 用途：显示当前工作目录的路径，让用户清楚知晓当前所在目录位置。

- 示例：在命令行中输入pwd，会输出类似/home/kali这样的当前目录绝对路径。

**cd：**

- 用途：用于切换当前工作目录。


- 语法及示例：
  - cd ..：返回上一级目录，其中..代表上一级目录，若要返回上两级目录，则可使用cd../..。
  - cd：单独使用时（在多数 Linux 系统默认配置下），返回用户主目录，通常是/home/用户名形式的目录。
  - cd /：切换到根目录，根目录是文件系统的起始点，用/表示。
  - cd -：返回上次操作目录，方便在两个目录间快速切换。

**ls：**和Windows里面的dir一样的

- 用途：用于列出目录内容，展示当前目录下有哪些文件和子目录。
- 语法及示例：
  - ls：列出当前目录下的文件和子目录，仅显示名称。
  - ls -l：以长格式列出文件和目录的详细信息，包括权限（如-rw-r--r--形式，分别表示文件所有者、所属组、其他用户对文件的读写执行权限）、所有者、大小（以字节为单位）、修改时间（精确到日期时间）等，便于深入了解文件特性。
  - ls -a：列出所有文件和目录，包括隐藏文件，隐藏文件通常以.开头，普通ls命令默认不显示它们。

**tree：**

- 用途：显示文件和目录的树形结构，以直观的层级方式呈现目录包含关系，能清晰看到各级子目录及其中的文件分布。
- 示例：在包含多层子目录和文件的目录下输入tree，会输出类似以树状分支形式展示的目录结构，根节点为当前目录，分支为子目录，叶子节点为文件。

**which：**

- 用途：查找某工具的完整路径，常用于确定系统命令在文件系统中的具体位置，方便排查命令执行问题或了解系统配置。
- 语法及示例：which nc：查看nc（Netcat 工具）的位置，执行后会输出nc工具所在的完整路径，如/usr/bin/nc。
- 注意：which只能用于显示 shell 命令的路径，不能用于非可执行文件或脚本，即它针对的是那些可以在命令行直接执行的程序，而非普通的数据文件、文本文件等。

 

**5、文件管理指令**

**touch：**

- 用途：用于创建空文件或更新文件的时间戳(访问和修改时间)。
- 语法：touch 文件名.后缀
- 示例：touch demo.txt，若demo.txt文件不存在，该命令会创建一个新的空文件；若文件已存在，它会更新文件的访问和修改时间。

**mkdir：**

- 用途：用于创建新的目录。

- 语法：mkdir 文件夹名称

**rmdir：**

- 用途：删除空目录。
- 语法：rmdir 文件夹名称

**rm：**

- 用途：用于删除文件或目录。
- 语法及示例：
  - rm  文件 ：删除名为指定的文件，如rm file.txt，删除file.txt文件。
  - rm -f 文件 ：强制删除文件，即使文件为只读等特殊状态也尝试删除。  
  - rm -r 文件夹  ：删除目录（文件夹）及其所有内容，-r表示递归删除，即逐层删除文件夹内的子文件夹及文件，直到为空。
  - rm -rf 文件夹 ：删除目录及其所有内容，-rf表示递归并强制删除，相比rm -r更 “强硬”，遇到权限问题等也强行处理。
  - 特别警示 ：rm -rf /* 是极其危险的操作，它会删除所有目录当中的文件，除非您完全了解其后果并确实打算这样做（在绝大多数情况下，这都不是一个好主意）。对于普通用户来说，这个命令应该被视为禁区。

**cp：**

- 用途：用于复制文件或目录。
- 复制文件语法：cp  旧名  新名
- 示例：cp password.txt pw.txt，将password.txt复制为pw.txt。
- 复制文件夹语法：cp  -r  旧名 新名
- 示例：cp  -r   A_work   B_work ，将A_work复制为B_work 。

**mv：**

- 用途：用于移动或重命名文件和目录。
- 语法及示例：
  - mv 旧名 新名：将文件或目录旧名重命名为新名，如mv A_name B_name，将A_name重命名为B_name。
  - mv 文件 目标地址：将文件移动到指定目标地址，如mv password.txt /home/kali/Documents/，将password.txt文件移动到/home/kali/Documents/目录下。

**echo：**

- 用途：写入内容，可覆盖，也可追加。
- 覆盖语法：echo   内容    > 文件
- 覆盖解析：echo命令用于输出文本或变量，>符号用于重定向输出，如果demo.txt文件已存在，此命令会覆盖文件中的原有内容；如果文件不存在，会创建该文件，如echo "Hello, World!" > demo.txt，将"Hello, World!"写入demo.txt，覆盖原有内容。
- 追加语法：echo   内容   >> 文件
- 追加解析：>>符号用于将输出追加到文件中，而不是覆盖文件内容。如果demo.txt文件不存在，会创建该文件，如echo "Another line of text." >> demo.txt，将"Another line of text."追加到demo.txt文件的末尾。

**cat：**

- 用途：查看文件内容。
- 语法：cat   文件
- 示例：cat demo.txt，显示demo.txt文件的内容。

**vim：**

- 用途：文件编辑器。
- 语法： vim  文件
- 操作：
  - i：进入编辑模式，可开始输入文本对文件进行编辑。
  - esc：退出编辑模式，回到命令模式，此时可进行保存、退出等操作。
  - :wq! + 回车：退出vim编辑器，wq表示保存并退出，!表示强制操作，当文件有一些特殊权限限制等情况时，可强制保存退出。

 

**6、压缩与解压**





请注意，在实际使用这些命令时，需根据具体的系统环境、需求以及文件特性来选择合适的压缩解压方式。例如，**如果对压缩比要求较高，可能优先考虑 bzip2 或 gzip 结合 tar 的方式；若只是简单归档，非压缩的 tar 包即可满足需求。



> **非压缩的 tar 包**不会做压缩  打包
>
> - 原理：
>   - tar（Tape Archive）命令主要用于将多个文件和目录打包成一个文件，它本身并不进行压缩操作。它只是简单地把文件和目录的元数据（如文件名、权限、时间戳等）和数据内容按照一定的格式组合在一起。
> - 特点：
>   - 优点：
>     - 打包速度快，因为没有压缩算法的处理过程，所以在处理大量文件时，tar可以迅速地将它们打包成一个文件。
>     - 对于归档存储，当不需要考虑磁盘空间问题，只需要将文件和目录集中保存时，tar包是很好的选择。
>   - 缺点：
>     - 由于没有压缩，文件大小不会减小。如果要存储或传输大量数据，可能会占用较多的磁盘空间和网络带宽。
>
> **bzip2 或 gzip 结合 tar 的方式**
>
> - 原理：
>   - 当tar与bzip2或gzip结合使用时，tar先将文件和目录打包，然后bzip2或gzip对打包后的文件进行压缩。
> - 特点：
>   - 优点：
>     - 可以大大减小文件大小。例如，对于一些日志文件、文本文件或者数据库备份文件等，经过bzip2或gzip压缩后，文件大小可能只有原来的几分之一甚至更小，节省了大量的磁盘空间和传输带宽。
>
>   - 缺点：
>     - 压缩和解压缩过程需要消耗更多的时间和计算资源。因为压缩算法要对数据进行复杂的处理，所以在处理大文件或者大量文件时，压缩操作可能会比较慢。
>
>
> 



bzip2：

- 用途：用于对**单个**文件进行压缩与解压操作，以实现文件存储优化或传输便利。
- 压缩：
  - 语法：bzip2 文件名
  - 示例：bzip2 demo.txt，此操作将创建一个名为demo.txt.bz2的压缩文件。
- 解压：
  - 语法：bunzip2 压缩文件名
  - 示例：bunzip2 demo.txt.bz2，执行该命令后将解压文件，恢复得到原始的demo.txt，同时压缩文件demo.txt.bz2会被删除。

gzip：

- ​	用途：同样专注于**单个**文件的压缩和解压，与 bzip2 类似但采用不同的压缩算法。
- 压缩：
  - 语法：gzip 文件名
  - 示例：gzip demo.txt，会创建一个名为demo.txt.gz的压缩文件，原文件demo.txt保留压缩版本。
- 解压：
  - 语法：gunzip 压缩文件名
  - 示例：gunzip demo.txt.gz，解压后还原为原始的demo.txt，demo.txt.gz文件消失。

rar（注意：非标准 Linux 工具）：

- 安装（仅适用于基于 Debian 的系统）：
  - 语法：apt-get install rar，通过此命令在基于 Debian 的系统上安装 rar 工具，以便后续进行压缩和解压操作。
- 压缩：
  - 语法：rar   a   压缩文件名.rar   目录
  - 示例：rar a 目录.rar 目录，这将把指定的 “目录” 文件夹压缩为 “目录.rar” 文件，方便存储和传输。
- 解压：
  - 语法：rar   x   压缩文件名
  - 示例：rar x 目录.rar，该操作将解压 “目录.rar” 文件到当前目录，还原出原始的文件夹结构和内容。

tar：

- 用途：功能强大，可用于创建不同格式的压缩包以及查看压缩包内容，常与其他压缩工具（如 bzip2、gzip）配合使用，尤其适用于处理文件夹的打包与解包。

- **创建一个非压缩的 tar 包（归档文件）：**
  - 语法：tar  -cvf  归档文件名.tar  目录  
  - 示例：tar -cvf  a.tar  fileone，此操作将创建一个名为a.tar的非压缩 tar 文件，包含 “fileone” 目录的内容，以方便文件归档与迁移。
  
- **取消一个非压缩的 tar 包（提取归档文件）：**
  - 语法：tar  -xvf  归档文件名.tar
  - 示例：tar  -xvf  a.tar  fileone，从a.tar 归档文件中提取 fileone 文件或录。
  
- **显示 tar 包内容：**
  - 语法：tar -tf 压缩文件名
  - 示例：tar -tf a.tar，将列出 “a.tar” 文件中的所有文件和目录，便于在不解包的情况下查看包内详情。
  
- **创建一个 bzip2 格式的压缩包：**
  - 语法：tar -jcvf  压缩文件名.tar.bz2  目录
  
  - 示例：tar -jcvf a.tar.bz2 fileone，会创建一个名为a.tar.bz2的 tar 和 bzip2 结合的压缩文件，包含 “fileone” 目录的内容，兼具 tar 的归档优势和 bzip2 的压缩性能。
  
- **解压 bzip2 格式的 tar 包：**
  - 语法：tar -jxvf 压缩文件名
  - 示例：tar -jxvf a.tar.bz2，将解压 “a.tar.bz2” 文件到当前目录，还原出原始的文件夹结构和内容。
  
- **创建一个 gzip 格式的压缩包：**
  - 语法：tar  -zcvf   压缩文件名.tar.gz   目录
  - 示例：tar -zcvf b.tar.gz fileone，创建一个名为 “b.tar.gz” 的 tar 和 gzip 结合的压缩文件，包含 “fileone” 的内容，利用 gzip 的压缩特性优化归档文件。
  
- **解压 gzip 格式的 tar 包：**
  - 语法：tar -zxvf 压缩文件名
  - 示例：tar -zxvf b.tar.gz，解压 “b.tar.gz” 文件到当前目录，恢复原始文件夹结构和内容。
